Technical information

signalml viewer requires Java 1.6.0+ compatible Java virtual machine. Although virtual machines by other publishers may work, only the Sun VM is officially supported and recommended. The JRE should be enough to run the application, the JDK distribution works just as well.

If you intend to run artifact detection or automatic sleep staging (analysis methods bundled with the application), you will also need Matlab Component Runtime v. 7.6. Appropriate binary install package is redistributed with this application.

Some parts of the application, notably the MP5 decomposition rely on a native application (mp5). Binary versions for Windows and generic Linux are included with the application. If you run this application on another operating system you need to (either) compile MP5 from sources yourself or use a server.

Known issues

  • Installation of MCR may fail on Windows if Visual C++ Redistributable is not installed. The problem manifests itself with a misleading error message that the instalation has been "interrupted". The Visual C++ Redistributable can be obtained freely from Microsoft here. Note that the same effect can also be caused by lack of administrative priviledges when MCR is installed.
  • MCR on x86_64 architecture causes error associated with the incorrect loading file.
